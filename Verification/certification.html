seL4 and Common Criteria EAL 7

The international standard known as Common Criteria (CC) focuses on computer security certification. Its highest Evaluation Assurance Level, EAL 7, requires some level of formal modelling and verification. The formal specifications and proofs provided by seL4 go beyond what is required at this highest level.

Namely, CC's EAL 7 requires a formal model of the security policy, of the functional specification and of the design specification, with formal correspondance between each. Then it only requires an informal mapping from the design specification to the implementation. In contrast, seL4 provides a formal proof to the implementation, and even further to the binary.

{% svg /images/tries-shutterstock/certif-cc-comparison.drawio.svg width="100%" %}

More precisely, formal methods are required in CC's assurance class for product development (CC contains 5 other assurance classes, such as tests, life-cycle support, or vulnerability assessment). As shown in the figure below, summarising the requirements for the developement class per EAL, the use of formal modelling and verification is only required from EAL 6, and only down to the design specification level.
